# UI/UX方針

## 1. 全体の方向性

### デザインテーマ
**ポップ × 洗練**：TikTok/Instagram のような躍動感と、婚活らしい品のある洗練さを両立。

### キーワード
- ときめき（心がざわつく体験）
- 軽やかさ（重くならない婚活）
- シェアしたくなる（バイラル設計）
- 温かみ（ロゴ・イラストに有機的な曲線）
- 発見（新しい出会いのワクワク感）

### 参考サービス
| サービス名 | 参考にする点 |
|------------|--------------|
| Hinge | カード型プロフィールの情報設計、コメントUI |
| Bumble | カラースキーム、フレンドリーなコピー |
| TikTok | フィードのスクロール体験、トレンド表示 |
| Notion | タイポグラフィの読みやすさ |

## 2. ビジュアル

### カラー方針
- メインカラー：ローズピンク `#F43F5E`（ときめき・情熱）
- サブカラー：コーラル `#FB7185`（明るさ・温かみ）
- アクセントカラー：ゴールド `#FBBF24`（特別感・ランキング）
- 背景：オフホワイト `#FFF1F2` / ホワイト `#FFFFFF`
- テキスト：チャコール `#1F2937`
- ボーダー・区切り：ライトピンク `#FCE7F3`

### タイポグラフィ方針
- 見出し：丸みのあるゴシック体（Noto Sans JP Bold）
- 本文：読みやすい標準ウェイト（Noto Sans JP Regular）
- 数字・英語：Inter（モダンで清潔感）
- 雰囲気：柔らかく親しみやすいが、ださくならない

### アイコン・イラスト
- Lucide React アイコン（細めのライン）
- ハート・桜・星などの婚活テーマのアクセントイラスト（SVG）
- プロフィール写真はアバター風のプレースホルダー

## 3. UX原則

### 重視する体験
1. **ときめきの演出**：マッチング成立時のアニメーション・エフェクトで感情を高める
2. **シェアしたくなるモーメント**：診断結果・マッチング成立はSNSシェアを自然に促す
3. **3タップ以内の達成感**：いいね・投稿・診断開始はすべて3タップ以内で完了

### 避けたいUX
1. 重たいオンボーディング（登録フォームが長すぎる）
2. ネガティブなフィードバック表示（「マッチングできません」など）

## 4. 画面構成

### 主要画面一覧
| 画面名 | 目的 | 主要要素 |
|--------|------|----------|
| ランディング | 新規登録促進 | キャッチコピー・診断CTA・登録ボタン |
| ホーム（タイムライン） | 投稿閲覧・交流 | フィード・いいね・コメント・投稿ボタン |
| 見つける（スワイプ） | マッチング | カード・スワイプUI・共通点バッジ |
| 診断 | エンタメ×自己理解 | 質問カード・結果ページ・シェアボタン |
| ランキング | 活況感演出 | 人気ユーザー・トレンドタグ |
| マイページ | プロフィール管理 | 写真・自己紹介・マッチング数 |

### 画面遷移図
```
[ランディング] → [登録] → [プロフィール設定] → [ホーム]
                                                    ↓
[ホーム（タイムライン）] ←→ [見つける（スワイプ）]
        ↓                           ↓
[投稿詳細]                    [マッチング成立] → [トーク]
        ↓
[診断] → [診断結果] → [シェア]
        ↓
[ランキング]
        ↓
[マイページ] → [プロフィール編集]
```

## 5. インタラクション

### アニメーション方針
- 積極的に使用（ときめき演出に不可欠）
- マッチング成立：ハートが弾けるパーティクルアニメ
- スワイプ：カードが傾いて飛ぶ動き
- いいね：ハートがぷよんと弾む

### フィードバック
- いいね送信：即座にハートアイコンが赤く染まる
- 投稿完了：「投稿しました！」トースト通知
- マッチング成立：全画面オーバーレイでお祝い演出

### エラー表示
- フォームバリデーション：フィールド下にインラインでピンク色のメッセージ
- ネットワークエラー：「通信に失敗しました。もう一度試してください」バナー
