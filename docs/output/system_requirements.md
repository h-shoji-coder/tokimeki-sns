# システム要件定義書 - ときめき（Tokimeki）

## システム概要

「ときめき」は婚活特化型SNSアプリである。従来の婚活マッチングアプリが抱える「飽きやすい・重い・拡散しない」という課題を、SNS的なタイムライン・シェアしたくなる恋愛診断・スワイプマッチングを組み合わせることで解決する。

1. **システムの目的**
   - 解決する課題：婚活疲れ・継続率の低さ・口コミ拡散の難しさ
   - 目標とする成果：楽しく続けられる婚活体験の提供、バイラル成長
   - 期待されるビジネスインパクト：フリーミアムモデルによるプレミアム転換収益

2. **主な機能**
   - コア機能：スワイプ型マッチング・タイムライン投稿・恋愛診断クイズ
   - 差別化機能：診断結果SNSシェア・共通点カウンター・マッチング演出アニメ
   - 付加価値機能：人気ユーザーランキング・トレンドハッシュタグ

3. **提供する価値**
   - ユーザー：重くなりがちな婚活を楽しく・毎日続けられる体験
   - ビジネス：バイラル拡散によるCAC低減、プレミアム課金収益
   - 社会的：少子化対策に資する出会いの場の提供

4. **実現方法の概要**
   - Next.js App Router によるSSR/CSR混在のモダンWebアプリ
   - Supabase（PostgreSQL）によるリアルタイムデータ管理
   - Clerk による認証
   - Vercel へのホスティング

## 情報の前提と仮定

- `docs/input/` フォルダ内の全ファイルを参照
- MVP段階ではメッセージ機能はモック（UIのみ）
- 決済機能はMVP外（プレミアムプランUIは表示するがモック）

## 機能要件

### 1. 認証・プロフィール
- 会員登録（メール / Google / LINE OAuth）
- プロフィール作成・編集（写真最大6枚・自己紹介・基本情報・趣味）
- プロフィール公開設定

### 2. スワイプ型マッチング
- 相手プロフィールのカード表示（写真・名前・年齢・共通点数・一言）
- 右スワイプ = いいね / 左スワイプ = パス
- 双方向いいねでマッチング成立
- マッチング成立時のアニメーション演出
- 1日10いいね制限（無料プラン）

### 3. 恋愛診断クイズ
- 10問・4択の選択式クイズ
- 恋愛タイプ判定（8種類）と詳細説明
- SNSシェア用カード画像生成（OGP対応）
- 相手との診断結果比較（相性スコア）

### 4. タイムライン（SNS）
- テキスト・画像の投稿作成
- いいね・コメント機能
- ハッシュタグ機能
- 人気投稿のトレンド表示

### 5. ランキング
- いいね数による人気ユーザーランキング（週次）
- 急上昇ハッシュタグ表示
- 今週のマッチング成立数（コミュニティ活況感）

### 6. 通知
- マッチング成立通知
- いいね・コメント通知
- プッシュ通知（将来対応）

---

## 非機能要件

- **パフォーマンス**：主要画面の初期表示3秒以内（スマホ4G環境想定）
- **セキュリティ**：Clerk による JWT 認証、個人情報の暗号化、18歳未満ブロック
- **スケーラビリティ**：初期1万ユーザー想定、Vercel+Supabase のオートスケール対応
- **可用性**：99.5%以上（Vercel SLA準拠）
- **アクセシビリティ**：WCAG AA準拠、スクリーンリーダー対応
- **レスポンシブ**：モバイルファースト（iOS Safari / Android Chrome 優先）

## 技術スタック

| レイヤー | 技術 | 選定理由 |
|----------|------|----------|
| フロントエンド | Next.js 15 (App Router) | SSR/ISR でSEO・OGP対応、診断シェアに必要 |
| 言語 | TypeScript 5.4+ | 型安全性・チーム開発効率 |
| スタイリング | Tailwind CSS v4 | 高速プロトタイピング・レスポンシブ対応 |
| 認証 | Clerk | SNSログイン対応、実装コスト低減 |
| データベース | Supabase (PostgreSQL) | リアルタイム機能・Row Level Security |
| ホスティング | Vercel | Next.js 最適環境、CDN・プレビュー機能 |
| アイコン | Lucide React | 軽量・デザイン一貫性 |

## リスク分析

| カテゴリ | リスク | 対策 |
|----------|--------|------|
| 技術的 | リアルタイム通信の負荷 | Supabase Realtime + 適切なチャンネル設計 |
| ビジネス的 | 男女比の偏り | 診断コンテンツで女性ユーザーを優先獲得 |
| 法的 | 出会い系規制法（出会い系サイト規制法） | 年齢確認機能の実装・利用規約整備 |
| 運用的 | 荒らし・スパム | 通報機能・自動フラグシステム |

- **ターゲットデバイス**：モバイルファースト（スマホ縦持ち最優先）
